  • 1.由于easyx本身库的限制，我们绘制在界面上的图片或者图形不可以删除，所以如果需要实现画面上某一组件的改变，需要重新绘制整个界面的所有内容。所以需要使用使用while(1)无限循环，检测某些状态并逐帧绘制图像，在符合某些条件时使用break跳出循环。
  • 2.使用while(1)逐帧绘图时，为了避免闪烁，需要用到在循环前使用BeginBatchDraw()函数用于开始批量绘图。执行后，任何绘图操作都将暂时不输出到绘图窗口上，直到执行 FlushBatchDraw 或 EndBatchDraw 才将之前的绘图输出。在循环内使用绘制完图片后使用FlushBatchDraw()函数用于执行未完成的绘制任务。跳出循环后使用EndBatchDraw()函数，结束批量绘制。具体实现原理可移步关于双缓冲消除闪屏的原理查看，此处不过多赘述。

它接受三个参数：

  • ExMessage msg：消息变量。
  • char* input：一个字符数组的指针，存储当前已经输入的文本。
  • int* length：一个指向整数的指针，表示当前 input 字符串的长度。

代码首先从 msg 中取出用户按下的字符，并将其存储在局部变量 c 中。

如果用户按下了退格键（ASCII码为 \b），并且 input 字符串长度大于0，则减少 length 的值，并在新的字符串末尾加上空字符（即移除了最后一个字符）。

如果按下的键既不是回车（\r）也不是退格（\b），则尝试将字符添加到 input 中。首先检查当前长度是否小于允许的最大长度 MAX_INPUT_LENGTH（这个值在代码中没有给出，但可以假设它是一个定义了的常量），如果是，则将字符 c 添加到当前长度的位置，随后增加 length 的值，并在新的末尾加上空字符。
